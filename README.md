# Traffic Vehicle Counting using YOLO-World

A computer vision project for traffic vehicle detection and ROI-based vehicle counting using **YOLO-World**. The project detects vehicles in a traffic intersection image and counts them inside manually defined Regions of Interest (ROIs). The application is fully containerized using **Docker**.

---

## Overview

This project uses the **YOLO-World** model to detect vehicles in a traffic intersection image and analyze vehicle distribution around pedestrian crosswalks.

Only **cars** and **trucks** are considered during the analysis.

---

## Objectives

The project performs the following tasks:

- Count the total number of vehicles in the image.
- Count vehicles behind the top crosswalk.
- Count vehicles inside multiple predefined ROI regions.
- Count vehicles inside each individual ROI:
  - Top ROI
  - Bottom ROI
  - Left ROI
  - Right ROI

---

## Approach

### Vehicle Detection

The project uses the pretrained **YOLO-World** model (`yolov8x-world.pt`) to detect objects in the image.

Only the following classes are used:

- Car
- Truck

All other detected objects are ignored.

---

### Vehicle Center Extraction

For each detected vehicle, the center point of its bounding box is calculated.

```python
cx = (x1 + x2) / 2
cy = (y1 + y2) / 2
```

---

### ROI Definition

Four polygon-based Regions of Interest (ROIs) are manually defined using **Shapely**:

- Top ROI
- Bottom ROI
- Left ROI
- Right ROI

---

### Vehicle Counting

A vehicle is counted inside an ROI if the center point of its bounding box lies inside the corresponding polygon.

## Docker

This project is fully containerized using Docker.

### Build the Docker image

```bash
docker build -t traffic-analysis-yolo .
```

### Run the container

```bash
docker run --rm traffic-analysis-yolo
```

This project uses the pretrained **YOLO-World** model:
